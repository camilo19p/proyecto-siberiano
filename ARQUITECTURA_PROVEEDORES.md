### Flow 2: Register Payment
```
User Click Payment Button ($)
    ?
Show Payment Modal
    ?
User Enter Amount
    ?
Validation (amount > 0)
    ?
Update Provider Deuda
    ?
Reset Dias en Mora (if deuda = 0)
    ?
Save to localStorage
    ?
Close Modal
    ?
Re-render Table
```

### Flow 3: Search & Filter
```
User Type Search
    ?
Update search state
    ?
useMemo calculates filtered list
    ?
Filter by name, NIT, or phone
    ?
Combine with estado filter
    ?
Calculate pagination
    ?
Re-render filtered table
```

## ?? Validations

```
Create/Update Provider:
??? Nombre: required (non-empty string)
??? NIT: required (non-empty string)
??? Teléfono: required (non-empty string)
??? Dirección: required (non-empty string)
??? Email: optional
??? Ciudad: optional
??? Deuda: optional (default 0)
??? Días: optional (default 0)

Payment:
??? Monto > 0: required
??? Monto <= current deuda: implicit (no validation, manual fix needed)

Delete:
??? User confirmation: required
```

## ?? Performance Considerations

```
Optimizations Implemented:
??? useMemo for filtered list (prevents recalc on every render)
??? Controlled inputs (avoid uncontrolled component warnings)
??? Event delegation (single handler for multiple elements)
??? localStorage batching (save on each action)

Potential Improvements:
??? Pagination in localStorage (if > 1000 items)
```
